System Architecture

AURA runs entirely in the browser. The web app communicates with the ESP32 over a local WiFi network via HTTP. AI inference (MediaPipe, TensorFlow.js) runs client-side. Only the NLP terminal makes an external call to OpenAI.

Any modern Chromium-based browser (Chrome, Edge, Brave). Camera features require getUserMedia permission. A secure context (HTTPS or localhost) is required for camera access.

Hardware Setup

Flash your ESP32 with firmware that exposes an HTTP endpoint for servo control. AURA sends angle commands using the following format:

http 
GET http://<ESP32_IP>/servo?s=1&angle=90

Parameters
  s      — servo index  (1 = Base, 2 = Shoulder, 3 = Elbow, 4 = Wrist, 5 = Gripper)
  angle  — target angle  0–180°

Wire each servo signal pin to a dedicated PWM GPIO on the ESP32. Wire servo power (V+ and GND) directly from the external power supply — never from the ESP32's onboard rails.

⚠ Servo motors draw significant current. Powering them from the ESP32's 3.3 V or 5 V pin can brown out the board. Use a separate BEC or bench supply with adequate current capacity.

Connecting ESP32

Settings → ESP32 Connection First-time setup
  1. Open AURA and navigate to the Settings tab (press 6 or click the tab)
  2. Under ESP32 IP Address, enter the local IP of your ESP32 (e.g. 192.168.1.42)
  3. Click Connect
  4. The status chip in the bottom bar turns green when the connection is live

Once connected, all slider, preset, gesture, vision, and terminal commands are forwarded to the physical arm in real time. Disconnecting reverts to simulator mode — the 3D twin continues to respond but no signals are sent to hardware.

Dashboard — Manual Control

Dashboard Sliders, Presets, Telemetry & Record/Replay key: 1

Full manual control over all five servos with real-time feedback.

  • Drag a slider or press − / + to move a servo in 1° increments
  • Preset buttons: Home (rest), Ready, Pick, Place
  • Press Record to capture a movement sequence, Stop to end, Play to replay
  • The live telemetry chart logs all five servo angles across the session
  • The 3D digital twin mirrors the arm's current angles in real time
  • The AI Trajectory Optimizer runs in the background and shows speed-boost % after sequences are recorded

Gesture Control

Gesture Hand Tracking via Google MediaPipe key: 2

Your webcam feed is processed entirely in the browser by MediaPipe. No server, no special hardware required.

  • Right hand X position → S1 Base left/right
  • Right hand depth (Z) → S2 Shoulder angle
  • Right hand Y position → S3 Elbow up/down
  • Left hand Y position → S4 Wrist rotation
  • Thumb–index pinch (right hand) → S5 Gripper open/close
ℹ Click Calibrate before starting gesture mode to set your neutral hand position. Good, even lighting and a plain background behind your hands improve tracking accuracy significantly.
// tab 3

Vision AI — Autonomous Object Sorter

Vision TensorFlow COCO-SSD Detection + Autopilot key: 3

COCO-SSD detects objects in the camera feed and instructs the arm to pick them up and deposit them into one of three bins.

  • Enter target classes (e.g. bottle, cup, apple) and set minimum confidence (default 55%)
  • Configure three drop zones: Bin A (Left, 145°), Bin B (Center, 90°), Bin C (Right, 35°)
  • Enable Fully Autonomous to auto-trigger pick-and-place on each detection
  • Enable Auto-Loop to continuously sort and cycle bins A → B → C
  • Human detection is a safety interlock — autopilot halts immediately if a person is detected in frame
  • Focus Zone controls how much of the frame the detector considers as the valid pickup area

Autopilot Pick-and-Place Sequence

① Align Base → ② Approach → ③ Grip → ④ Lift → ⑤ Deliver → ⑥ Release → ⑦ Home
// tab 4

Face Scan

Face Scan Head Pose → Servo Mapping key: 4

Facial landmarks are used to map head pose directly to servo positions. Only three servos are mapped in this mode.

  • Tilt head left / right → S1 Base rotation
  • Nod head up / down → S3 Elbow angle
  • Open / close mouth → S5 Gripper open/close

Click Calibrate with your head in a neutral, forward-facing position before controlling the arm.

// tab 5

AI Command Terminal

Terminal Natural Language Control via GPT-4o-mini key: 5

Type any plain-English instruction. GPT-4o-mini interprets it and executes the matching servo command sequence.

  • "pick up the object" → full pick sequence
  • "move left" → rotates S1 Base counterclockwise
  • "go to ready position" → Ready preset
  • "grab and place in bin A" → pick-and-place to Bin A (145°)
  • "open gripper" → S5 to maximum open
  • "elbow up" → increments S3
⚠ The terminal requires an OpenAI API key, entered in Settings → Terminal. Each command makes an API call billed to your OpenAI account. Commands that may conflict with the current safety state (e.g. commanding movement while E-Stop is active) are rejected.

Source Code Guide

AURA is a single-page application. Each control mode lives in its own JavaScript module. All modules share a central servo state object and call the same dispatch function.

aura/
  index.html           — entry point; loads all tabs and bootstraps the app
  css/
    style.css          — global layout, tokens, theme classes (cyber / solar / matrix)
  js/
    servo.js           — servo state, angle clamping, HTTP dispatch to ESP32
    dashboard.js       — slider controls, preset buttons, record/replay engine
    gesture.js         — MediaPipe hand tracking loop, gesture→servo mapping
    vision.js          — COCO-SSD inference loop, autopilot state machine
    face.js            — facial landmark detection, head-pose→servo mapping
    terminal.js        — GPT-4o-mini API calls, response→action parser
    twin.js            — Three.js scene, arm model, per-frame angle update
    optimizer.js       — ML trajectory model, speed-boost analysis, analytics
    settings.js        — ESP32 config, camera device select, theme toggle
    keyboard.js        — global keydown handler, shortcut dispatch table
    gamepad.js         — Gamepad API polling loop, axis→servo mapping
    telemetry.js       — live chart, session stats, command counter

Sending a Servo Command

Every control mode calls the same central dispatch function in servo.js:

javascript — servo.js 
// Central servo dispatch — called by all control modes
async function setServo(index, angle) {
  angle = Math.max(0, Math.min(180, Math.round(angle)));

  servoState[index] = angle;        // update shared state
  update3DTwin(index, angle);        // sync Three.js model
  updateTelemetry(index, angle);     // log to chart
  updateSlider(index, angle);        // reflect in dashboard UI

  if (eStopActive) return;           // hard block during E-Stop

  if (esp32Connected) {
    await fetch(
      `http://${esp32IP}/servo?s=${index}&angle=${angle}`
    );
  }
}

Share / Remote Access

The Settings tab exposes a shareable link. Anyone with the link and network access to the same ESP32 can control the arm remotely — useful for demos and collaborative testing. The link encodes the ESP32 IP and session token as URL parameters.

Troubleshooting

Issue ESP32 won't connect
  • Confirm your ESP32 and browser are on the same WiFi network (same SSID/LAN)
  • Find the correct IP via the Arduino Serial Monitor or your router's DHCP table
  • AURA is served over HTTPS; browsers block mixed-content HTTP requests by default. Either access AURA over plain HTTP or configure your ESP32 with SSL (unusual for local dev)
  • Try ping <esp32_ip> from the same machine to confirm reachability
Issue Camera permission denied
  • Allow camera access when the browser prompts you on first load
  • In Chrome, go to chrome://settings/content/camera and allow aurasoftware.vercel.app
  • Camera access is only granted in secure contexts — HTTPS or localhost
  • If using a VM or remote desktop, ensure the virtual camera is passed through
Issue Arm moves erratically / jitters
  • Check power supply. Servos drawing too much current cause voltage drops.
  • Ensure common ground between ESP32 and external power supply.
  • In Gesture mode, ensure your hand is clearly visible and lighting is good.
